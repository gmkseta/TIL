# 테스트 대역

* 운영 코드를 대신하는 코드의 대역

### 대역과 가정

* DOC 준비 비용이 큰 경우
  * 구동에 많은 자원이 필요
  * 환경 제어가 어려움
* DOC가 SUT - (System Under Test)에 제공하는 계약을 준수하는 대역 코드를 사용
* 대역 코드가 계약을 DOC와 동일하게 준수할 것이라고 가정 - DOC를 그대로 구현하는건 아니니까
